Krátce:

document.addEventListener('DOMContentLoaded', ...)
Spustí se jakmile je hotový DOM (HTML je zparsované a synchronní/defer skripty jsou dojeté). Nečeká na obrázky, iframy, fonty apod. Hodí se, když chceš hned pracovat s elementy.

window.addEventListener('load', ...)
Spustí se, až je všechno načtené: DOM i veškeré resource (obrázky, CSS, fonty, iframy…). Je později než DOMContentLoaded.

window.onload = ...
Stejné načasování jako window.addEventListener('load'), ale je to přiřazení vlastnosti. Můžeš mít jen jeden handler (poslední přiřazení přepíše předchozí). addEventListener jich umí víc.

Přesnější rozdíly
Věc	DOMContentLoaded	window 'load' (addEventListener/onload)
Kdy se spustí	Po zparsování HTML a dokončení synchronních a defer skriptů	Po načtení všech zdrojů (obrázky, iframy, CSS, fonty…)
Čeká na obrázky?	Ne	Ano
Čeká na defer skripty?	Ano	Ano (a navíc i na ostatní resource)
Čeká na async skripty?	Ne (běží nezávisle)	Ne (ale často doběhnou dřív než load)
Více handlerů	Ano (addEventListener)	Ano (addEventListener), ne u window.onload (jen jeden)
Typické použití	Iniciace UI, selektory, navázání listenerů	Věci závislé na obrázcích/rozměrech, měření, lazy init po vše načteno

Pozn.: Samotné CSS bývá render-blocking, ale nečeká se na něj pro DOMContentLoaded (ten může být nepřímo zpožděn, pokud CSS blokuje skripty, které blokují parser). Prakticky: DOMContentLoaded = „DOM hotový“, load = „všechno hotové“.

Doporučené vzory

Nejčastěji stačí DOMContentLoaded:

document.addEventListener('DOMContentLoaded', () => {
  initUI();
});


Potřebuješ mít jistotu, že jsou načtené obrázky (např. měření rozměrů):

window.addEventListener('load', () => {
  measureGalleryLayout();
});


Nikdy nepřepisuj onload, pokud přidáváš víc handlerů:

// špatně – poslední přepíše předchozí
window.onload = fn1;
window.onload = fn2;

// správně
window.addEventListener('load', fn1);
window.addEventListener('load', fn2);


S defer skripty (doporučeno u <script src> v <head>):
defer zajistí, že script běží po parsování DOMu, takže často není potřeba čekat na DOMContentLoaded.

<script src="/app.js" defer></script>


Chceš-li, pošli kousek kódu a řeknu, které z těch tří použít (a proč) přímo ve tvém případě.
